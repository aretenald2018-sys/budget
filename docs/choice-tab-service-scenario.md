# 선택 탭 재설계 - Choice OS 서비스 시나리오

## 1. 제품 정의

`선택` 탭은 사고 싶은 것과 하고 싶은 것을 즉시 실행하지 않도록 막는 금욕 앱이 아니다.

핵심은 **욕망을 없애지 않고 좋은 타이밍까지 보관하고, 조건이 맞으면 퀘스트처럼 실행하며, 넘기거나 덜어낸 경험은 감각뱅크에 축적하는 것**이다.

한 문장으로 정의하면:

> 지금 끌리는 것을 바로 소비하지 않고, 조건과 시간과 기록을 붙여 더 나은 선택으로 숙성시키는 개인 선택 운영체제.

## 2. 핵심 원칙

1. **오늘 할 수 있는 것부터 보여준다.**
   - 첫 화면의 주인공은 전체 목록이 아니라 `오늘 열린 선택`이다.
   - 사용자가 앱에 들어온 순간 답해야 하는 질문은 "내가 뭘 담아뒀지?"가 아니라 "오늘 무엇을 실행할 수 있지?"다.

2. **구매와 행동을 같은 문법으로 다루되 의미는 구분한다.**
   - 사고 싶은 옷은 목표가 아니라 `보류된 선택지`다.
   - "2주 예산이 5만원 남으면 그 옷 사기"가 되는 순간 `조건부 퀘스트`가 된다.
   - 저장한 레시피, 방문하고 싶은 장소, 루틴도 같은 카드 문법을 쓰되 타입과 조건으로 다르게 읽히게 한다.

3. **조건은 행동을 허락하는 문이다.**
   - 날짜, 반복 요일, 예산 잔액, 몸무게, 장소 방문, 수동 체크를 모두 조건으로 지원한다.
   - 조건은 사용자가 직접 고를 수 있어야 한다.

4. **실패라는 단어를 쓰지 않는다.**
   - 결과 액션은 `실행함`, `넘김`, `보류 연장`, `삭제`로 둔다.
   - 사용자가 샀거나 먹었거나 건너뛴 것은 모두 기록 가능한 선택이다.

5. **감각뱅크는 별도 목적지가 아니라 결과 레이어다.**
   - 감각뱅크는 탭 안에서 독립 섹션으로 보이지만, 모든 선택과 퀘스트의 결과가 흘러드는 축적 시스템이다.
   - 와인, 야식, 구매 충동, 루틴 성공 등에서 절약 금액, 칼로리, 잔 수, 횟수, 연속일, 기분을 적립한다.

6. **게임화는 귀엽지만 가볍지 않아야 한다.**
   - XP, 레벨, 연속 기록, 배지, 퀘스트 로그를 지원한다.
   - 다만 금융 앱 안에 있으므로 색과 모션은 절제한다.
   - 컨페티와 축하는 조건이 열린 순간 또는 완료 순간처럼 드문 이벤트에만 쓴다.

7. **선택 탭은 관리 도구보다 이미지 피드에 가까워야 한다.**
   - 사용자가 사고 싶거나 해보고 싶은 것은 먼저 이미지로 기억된다.
   - 첫 화면은 무신사 추천 랜딩처럼 큰 비주얼 캐러셀과 상품/경험 카드가 주도한다.
   - 텍스트 목록은 보조이고, 이미지는 보류함의 주요 기억 단서다.
   - 다만 목적은 구매 전환이 아니라 `기다림`, `조건`, `진행도`를 얹어 좋은 타이밍까지 보관하는 것이다.
   - 파싱한 이미지가 예쁘지 않거나 비었을 때는 사용자가 `원본 썸네일 / 앱 생성 비주얼 / 무료 사진 후보 / 직접 이미지 URL` 중에서 교체할 수 있어야 한다.
   - 각 카드의 이미지 영역은 직접적인 진입점이다. 이미지를 누르거나 `이미지 추천`을 누르면 제목 기반 검색어가 자동으로 들어간 교체 모달이 열린다.
   - 구매 후보뿐 아니라 조건 카드로 전환된 퀘스트도 같은 이미지 교체 흐름을 가져야 한다.
   - 보류함의 모든 카드는 상세/수정 시트를 가져야 한다. 제목 영역이나 `상세`를 누르면 이미지, 제목, 가격, 메모, 실행/보관 상태를 바로 고칠 수 있어야 한다.
   - 상세 시트 안에서 `구매 조건`을 바로 만들 수 있어야 한다. 예: 현재 확보액과 구매 허용액을 입력하고 `이 조건으로 보류`를 누르면 조건 진행 중으로 전환된다.
   - 예: `일본가기`는 `일본 여행`, `뱃살 쏙 빠지는 샐러드`는 `샐러드` 후보를 우선 보여준다.

## 3. 정보 구조

```
선택 탭
├─ 오늘
│  ├─ 오늘 열린 선택 / 퀘스트
│  ├─ 거의 열린 선택
│  ├─ 보류 중인 선택
│  └─ 감각뱅크 요약
├─ 보류함
│  ├─ 구매
│  ├─ 경험
│  ├─ 루틴
│  ├─ 방문
│  └─ 감각
├─ 감각뱅크
│  ├─ 절약 금액
│  ├─ 절약 칼로리
│  ├─ 줄인 잔 수
│  ├─ 좋은 선택 횟수
│  ├─ 연속일
│  └─ 기분 기록
└─ 기록
   ├─ 실행함
   ├─ 넘김
   ├─ 보류 연장
   └─ 삭제됨
```

상단 세그먼트는 `오늘 / 보류함 / 감각뱅크` 3개를 기본으로 한다.

`오늘`은 기본 진입점이다. `보류함`은 전체 저장소이고, `감각뱅크`는 성취와 절제의 축적판이다.

## 4. 핵심 객체

### 4-1. Choice

사용자가 끌림을 저장한 기본 단위.

```ts
type ChoiceType = 'purchase' | 'experience' | 'routine' | 'visit' | 'sensory' | 'manual';

interface Choice {
  id: string;
  title: string;
  type: ChoiceType;
  note?: string;
  sourceUrl?: string;
  imageUrl?: string;
  originalImageUrl?: string;
  visualMode?: 'auto' | 'original' | 'generated' | 'stock' | 'custom';
  visualCredit?: string;
  estimatedPrice?: number;
  estimatedKcal?: number;
  estimatedAlcoholGlasses?: number;
  waitPolicy: WaitPolicy;
  condition?: Condition;
  status: 'waiting' | 'almost' | 'ready' | 'done' | 'skipped' | 'extended' | 'archived';
  createdAt: string;
  readyAt?: string;
  completedAt?: string;
}
```

### 4-2. WaitPolicy

```ts
type WaitPolicy =
  | { type: '24h' }
  | { type: 'cycleEnd' }
  | { type: 'date'; date: string }
  | { type: 'condition' };
```

사용자는 저장 시 아래 중 하나를 고른다.

- 24시간만 기다리기
- 이번 2주 끝까지 기다리기
- 날짜 직접 지정
- 조건 충족 때까지

### 4-3. Condition

```ts
type Condition =
  | { type: 'date'; date: string }
  | { type: 'weekday'; weekday: number; timeOfDay?: string; recurrence: 'weekly' | 'biweekly' | 'monthly' }
  | { type: 'budgetRemaining'; cycle: 'biweekly' | 'monthly'; amount: number; categoryId?: string }
  | { type: 'bodyMetric'; metric: 'weight' | 'bodyFat' | 'sleep' | 'steps'; operator: '<=' | '>='; value: number; unit: string }
  | { type: 'location'; placeName: string; radiusM?: number }
  | { type: 'routine'; label: string; count: number; window: 'week' | 'month' }
  | { type: 'manual'; label: string };
```

처음부터 모든 조건 타입을 지원 대상으로 둔다.

우선순위는 `date`, `weekday`, `budgetRemaining`, `bodyMetric`, `manual`을 먼저 안정화하고, `location`은 권한과 배터리 이슈 때문에 별도 권한 플로우가 필요하다.

## 5. 상태 모델

```
waiting  -> almost  -> ready  -> done
    |         |          |        |
    |         |          |        -> bank entry 생성 가능
    |         |          -> skipped / extended
    |         -> condition progress 80% 이상
    -> archived
```

- `waiting`: 아직 시간이 남았거나 조건이 멀다.
- `almost`: 조건이 80% 이상 찼다. 홈에서는 아직 띄우지 않고 선택 탭 안에서만 조용히 보여준다.
- `ready`: 오늘 실행 가능하다. 선택 탭 첫 화면 최상단에 오른다.
- `done`: 실행했다.
- `skipped`: 넘겼다. 감각뱅크 적립 후보가 된다.
- `extended`: 보류 기간을 늘렸다.
- `archived`: 사용자가 더 이상 관리하지 않는다.

## 6. 첫 화면 구성

### 6-1. 앱바

- 좌측: `선택`
- 보조 pill: `Lv. 7 · 연속 5일`
- 우측: 검색, 설정

게임화는 앱바에서 작게만 보여준다. 첫 화면을 점수판으로 만들지 않는다.

### 6-2. 오늘 요약 카드

첫 화면 상단은 요약 카드보다 **비주얼 캐러셀**이 우선한다.

캐러셀은 3-5초마다 자동으로 스와이프되며, 사용자가 담아둔 구매/경험/루틴 이미지를 큰 이미지로 보여준다.

캐러셀 구성:

- 이미지: 상품, 레시피, 장소, 와인, 경험 이미지
- 상태 pill: `24시간 대기 완료`, `목요일 저녁 열림`, `예산 조건 84%`
- 큰 카피: 사용자가 왜 이 선택을 저장했는지 감정적으로 떠올릴 수 있는 문장
- 보조 카피: 어떤 조건으로 열렸는지
- 하단 progress dot: 3-5초 자동 전환 진행도

라이트 모드 화면 안에서 캐러셀 자체는 이미지가 주도하되, 카드/피드의 UI chrome은 기존 라이트 토큰을 따른다.

표시 정보:

- 오늘 열린 선택 수
- 이번 2주 절약/보류 금액
- 감각뱅크 증가량
- 오늘 XP

카피 예시:

- `오늘 3개의 선택이 열렸어요`
- `바로 사지 않은 덕분에 84,000원이 아직 남아있어요`

### 6-3. 오늘 열린 선택

가장 중요한 섹션.

카드는 관리 리스트가 아니라 2열 이미지 피드 카드가 기본이다.

카드 안에는 반드시 다음이 보여야 한다.

- 타입: 구매, 경험, 루틴, 방문, 감각
- 대표 이미지
- 제목
- 왜 열렸는지
- 실행했을 때의 보상 또는 의미
- 목표까지 얼마나 남았는지 보여주는 gauge
- 액션: `실행함`, `넘김`, `보류 연장`

### 6-4. 거의 열린 선택

조건이 80% 이상 찬 항목.

예:

- `예산 42,000원 남음 · 50,000원까지 8,000원`
- `몸무게 73.1kg · 목표 72.0kg`

거의 열린 선택은 강하게 축하하지 않는다. 기대감을 주되 행동을 재촉하지 않는다.

거의 열린 선택도 이미지 카드로 표현한다. 각 카드에는 `기다림`, `예산`, `몸무게`, `장소`, `루틴` 중 어떤 조건이 진행 중인지와 진행률 gauge를 둔다.

### 6-5. 보류 중

장바구니 효과를 담당한다.

표시 정보:

- 제목
- 가격 또는 감각 비용
- 대기 정책
- 남은 시간
- 연결 조건

구매 CTA는 바로 노출하지 않는다. 구매 링크는 상세 안에 둔다.

### 6-6. 감각뱅크 요약

첫 화면 하단 또는 상단 요약의 일부로 들어간다.

표시 지표:

- 절약 금액
- 절약 칼로리
- 줄인 잔 수
- 좋은 선택 횟수
- 연속일
- 최근 기분

## 7. 생성 플로우

새 선택 추가는 하단 sheet로 연다.

상품/장바구니형 선택은 기존 이미지 파싱 기능을 적극 활용한다.

사용자가 쇼핑몰 이미지, 상품 상세 캡처, 공유 이미지, 릴스 썸네일을 붙이면:

- 대표 이미지
- 예상 상품명
- 예상 가격
- 카테고리
- 출처/링크
- 감각뱅크 연결 후보

를 먼저 보여주고, 그 다음 기다림 방식과 조건을 고른다.

### Step 1. 무엇을 담을지 선택

타입:

- 구매
- 경험
- 루틴
- 방문
- 감각

### Step 2. 기다림 방식 선택

- 24시간
- 이번 2주 끝
- 날짜 지정
- 조건 충족

### Step 3. 조건 선택

조건 후보:

- 날짜/요일
- 예산 잔액
- 몸무게/건강 수치
- 장소 방문
- 반복 루틴
- 수동 체크

### Step 4. 감각뱅크 연결

선택 사항:

- 넘기면 절약 금액 적립
- 넘기면 칼로리 적립
- 음주 잔 수 적립
- 좋은 선택 횟수 적립
- 연속일 갱신
- 기분 기록

## 8. 감각뱅크 통합 방식

감각뱅크는 별도 세그먼트로 둔다. 하지만 데이터는 모든 선택 결과에서 생성된다.

### 적립 이벤트

1. 구매 선택지를 `넘김`
   - estimatedPrice가 있으면 절약 금액에 적립
2. 음식/와인 선택지를 `넘김`
   - estimatedPrice, estimatedKcal, estimatedAlcoholGlasses 적립
3. 루틴을 `실행함`
   - 좋은 선택 횟수, XP, streak 적립
4. 보류를 `연장`
   - 선택 유지 streak 적립

### 감각뱅크 카드 문법

- `금액`: 너무 크고 자극적으로 만들지 않는다. "절약한 돈"보다 "아직 내 편인 돈"에 가깝게 표현한다.
- `칼로리`: 죄책감 언어를 쓰지 않는다. "덜어낸 열량" 정도로 표현한다.
- `술`: "안 마신 잔"보다 "미룬 잔" 또는 "줄인 잔"이 중립적이다.
- `기분`: 사용자의 자책을 막기 위해 좋음/나쁨보다 "가벼움, 안정, 아쉬움, 충동 높음" 같은 상태 태그를 쓴다.

## 9. 게임화 시스템

### XP

- 선택 저장: +5 XP
- 24시간 보류 성공: +10 XP
- 조건부 퀘스트 실행: +25 XP
- 넘김 기록: +20 XP
- 감각뱅크 기록: +10 XP
- 연속일 유지: +5 XP 보너스

### 레벨

레벨은 사용자의 자제력을 평가하는 점수가 아니라 기록의 누적량이다.

카피:

- `Lv. 7 선택 수집가`
- `다섯 번째 좋은 선택`
- `이번 2주 84 XP`

### 배지

- `24시간 장인`: 24시간 보류 10회
- `목요일 요리사`: 요일 루틴 4회
- `예산 수문장`: 2주 예산 조건 3회 달성
- `감각 저금가`: 감각뱅크 10만원
- `한 잔 미루기`: 와인/음주 줄임 5회

### 컨페티

이번 범위에서는 홈 팝업은 미구현이다.

하지만 설계상 이벤트는 남겨둔다.

- `ready`가 처음 된 순간
- `done`이 된 순간
- streak milestone 달성
- 감각뱅크 milestone 달성

## 10. 서비스 시나리오

### 시나리오 A. 사고 싶은 옷

1. 사용자가 데님 셔츠 링크를 공유한다.
2. 선택 탭이 `구매` 타입으로 저장한다.
3. 사용자는 기다림 방식을 `이번 2주 끝` 또는 `예산 잔액 50,000원 이상`으로 고른다.
4. 오늘 화면에는 `거의 열린 선택`으로 보인다.
5. 조건이 열리면 `오늘 열린 선택`으로 올라온다.
6. 사용자는 `실행함`, `넘김`, `보류 연장` 중 하나를 고른다.
7. 넘기면 감각뱅크에 금액이 적립된다.

### 시나리오 B. 목요일 레시피 퀘스트

1. 사용자가 레시피 Shorts/Reels 링크를 저장한다.
2. 타입은 `루틴`, 조건은 `매주 목요일 저녁`.
3. 목요일 저녁이 되면 오늘 화면 최상단에 열린다.
4. 완료하면 XP, streak, 요리 배지가 쌓인다.

### 시나리오 C. 와인 미루기

1. 사용자가 와인이 끌린다고 기록한다.
2. 선택은 `감각` 타입으로 저장된다.
3. 오늘 마시지 않거나 양을 줄이면 `넘김`으로 기록한다.
4. 감각뱅크에 금액, 칼로리, 줄인 잔 수, 기분이 쌓인다.

### 시나리오 D. 장소 방문

1. 사용자가 `롯데마트 가면 장보기`를 저장한다.
2. 타입은 `방문`, 조건은 `장소 방문`.
3. 위치 권한이 허용된 경우 장소 근처에서 ready가 된다.
4. 권한이 없으면 수동 체크로 대체한다.

### 시나리오 E. 몸무게 조건 보상

1. 사용자가 `72kg 도달하면 러닝화 구매`를 저장한다.
2. 타입은 `구매`, 조건은 `몸무게 <= 72kg`.
3. 수치가 가까워지면 `거의 열린 선택`에 표시한다.
4. 도달하면 오늘 화면에 올라온다.

## 11. 화면 톤

### 색

기존 홈 디자인 시스템의 라이트 토큰을 따른다.

- 기본 배경: `#f5f6fa`
- 카드: `#ffffff`
- 텍스트: `#191f28`
- 보조 텍스트: `#4e5968`
- 힌트: `#8b95a1`
- accent: `#6366f1`
- progress: `linear-gradient(90deg, #7a8cff, #b388ff)`

선택 탭은 게임화가 들어가지만 전체 화면을 보라색 게임 UI로 만들지 않는다. 보라는 진행, 활성, 보상 순간에만 얇게 쓴다.

### 이미지 피드

- 첫 화면은 큰 자동 캐러셀 + 작은 썸네일 rail + 2열 이미지 카드로 구성한다.
- 캐러셀은 3-5초마다 자동으로 다음 이미지로 넘어간다.
- 피드 카드는 상품 판매 UI처럼 이미지를 크게 보여주되, CTA는 구매가 아니라 `실행함`, `넘김`, `연장`, `조건 보기`가 된다.
- 이미지 위에는 상태 pill과 가격/XP/조건 요약만 얹는다.
- 텍스트 설명은 카드 하단에 짧게 둔다.
- 모든 카드에는 최소 1개의 gauge를 둔다. 예: 기다림 24/24h, 예산 84%, 몸무게 72%, 장소 접근 68%, 루틴 3/4.

### 이미지 파싱

- `보류함에 담기` sheet의 첫 컴포넌트는 텍스트 입력이 아니라 이미지 파싱 결과 card다.
- 파싱 완료 상태는 이미지 위 scan frame과 confidence pill로 보여준다.
- 파싱된 가격/카테고리/기본 기다림/감각뱅크 연결 후보를 작은 detection tile로 보여준다.
- 사용자는 파싱 결과를 확인한 뒤 기다림 방식과 조건을 고른다.

### 이미지 fallback 계층

이미지가 비어 있을 때도 회색 빈칸을 보여주지 않는다. 선택 탭은 이미지가 기억 단서이므로 fallback도 상품/경험 카드처럼 보여야 한다.

우선순위:

1. 원본 페이지 `og:image`, 상품 이미지, YouTube/Shorts/Reels 썸네일
2. 사용자가 업로드/공유한 캡처 이미지
3. 무료 스톡 검색 API 결과
   - 예: Pexels, Pixabay
   - 반드시 서버에서 검색하고 결과를 캐시한다.
   - 출처/사진가 표기 정책을 따른다.
4. 이미지 생성 API 결과
   - 예: `피코 데 가요 레시피`처럼 원본 숏츠 썸네일보다 더 매력적인 editorial food/product visual이 필요할 때 사용한다.
   - 비용이 있으므로 사용자가 명시적으로 켜는 옵션으로 둔다.
5. CSS/SVG generated visual
   - API 없이 무료로 쓸 수 있는 최후 fallback.
   - 음식, 패션, 와인, 운동, 장소 타입별 gradient/shape를 다르게 만든다.

무료 우선 MVP에서는 1, 2, 5를 먼저 구현한다. 3은 무료 API key와 attribution UI가 준비되면 추가한다. 4는 비용과 서버 보안이 준비된 뒤 선택 옵션으로 둔다.

### 카드

- 카드 반경: 16px
- 내부 padding: 16px 18px
- 일반 카드 shadow: `0 2px 8px rgba(15,23,42,.06), 0 1px 3px rgba(15,23,42,.04)`
- 카드 안 카드 중첩 금지

### 밀도

첫 화면의 정보 우선순위:

1. 오늘 실행 가능한 것
2. 거의 열린 것
3. 보류 중인 것
4. 감각뱅크 요약

보류함과 기록은 깊은 화면에 둔다.

사용하는 말:

## 13. 구현 단계

### Phase 1. 화면 재구성

- 선택 탭 첫 화면을 `오늘 / 보류함 / 감각뱅크` 구조로 정리
- 기존 cart item, pact, mindbank entry를 새 카드 언어로 매핑
- 홈 팝업은 미구현

### Phase 2. 조건 엔진

- 날짜, 요일 반복, 예산 잔액, 수동 체크
- body metric은 수동 입력 기반
- location은 권한 플로우 준비 후 적용

### Phase 3. 게임화

- XP
- streak
- badge
- 퀘스트 로그

### Phase 4. 감각뱅크 통합

- 구매, 음식, 와인, 루틴 결과에서 bank entry 생성
- 기존 wine/mindbank 데이터는 새 `BankEntry`로 마이그레이션
