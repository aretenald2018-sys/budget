# 설정 화면 정의서 (재설계)

> **[2026-07-24 개정] 목업 SSOT 재설계로 대체됨.** 사용자가 제공한 "설정 각 항목 화면"
> 목업(10개 독립 화면)이 디자인 SSOT로 확정되어, 본 계약의 §2 재구성안과 §0 중
> "보상 폼 설정 제거" 결정은 `docs/ai/flows/2026-07-24-settings-10-screens.md`가 대체한다
> (포인트/미션은 설정 05 화면으로 신설). 얇은 허브 + drill-in 모달 원칙 자체는 유지.

> 화면 하나하나 설계(기획자 화면정의서) 첫 대상. 현재 설정 화면이 "엉망"이라는
> 사용자 지적에 따라 **미니멀·컴팩트**로 다시 설계한다.
> 원칙: 설정 화면은 **얇은 허브** — 최상위는 한 줄 행 목록, 세부 폼은 전부 drill-in(하위 화면/모달).
> 시각 토큰은 `docs/design-system.md`. 갈리는 결정은 §5 객관식으로 남기고 사용자가 답한다.

## 0. 확정 결정 (2026-07-24)

- 세부 폼 drill-in = **모달** (기존 모달 인프라 재사용).
- 바로가기(검토 대기·정산·월간 리포트) = **설정에 유지** (정산은 설정이 유일 진입점).
- **설정 화면에서 제거** (기능 코드·데이터 경로는 보존, 설정 UI만 정리):
  - DayBird 연동 패널 — 설정에만 존재하므로 사실상 폐지.
  - Android 알림 수집 패널 — **기능·APK 경로는 그대로**, 설정 패널만 제거.
  - 보상 적립 대형 폼 — 자동 적립 로직·기존 포인트 데이터는 건드리지 않고 설정 폼만 제거.
- 분리 처리: "오늘 카드를 홈에서 그날 포인트 유도"는 포인트 시스템을 자동→수동으로 바꾸는 새 기능이라
  별도 구상 세션('홈 오늘 카드')으로 다룬다. 이번 설정 슬라이스 범위 밖.

## 1. 현재 상태 진단 (실물 검수)

- `?fixture=basic` 설정 탭 실측: **세로 4551px** (뷰포트의 ~12배). 스크롤 지옥.
- 한 화면에 7개 대형 섹션의 **세부 폼이 전부 인라인**으로 펼쳐짐:
  1. 프로필 + 로그아웃
  2. DayBird 대시보드 연결 (큰 카드)
  3. 예산 & 카테고리 (월 예산 요약 + **카테고리 예산 전체 리스트[매우 김]** + 홈 관리 카테고리 피커)
  4. 충당금 (비상금 주머니)
  5. 화면 & 소계획 (테마 세그먼트 + **보상 적립 대형 폼 — 필드 10+개**)
  6. 계좌 & 데이터 소스 (검토/정산/리포트 바로가기 3 + Android 수집 + 정산 규칙 폼 + 규칙 리스트)
  7. 앱 정보 (버전 + APK 다운로드)
- 밀도 불일치: 요약 카드 / 조밀한 폼 / 네비 행이 한 화면에 뒤섞여 위계가 안 보임.

## 2. 재구성안 (미니멀 허브)

최상위 = 한 줄 행. 상세는 drill-in. 목표: 4551px → 1~1.5 스크린.

| 그룹 | 최상위에 남길 것 (행) | drill-in으로 밀 것 |
| --- | --- | --- |
| 계정 | 프로필 + 로그아웃 | — |
| 예산 | 월 예산 요약 지표(총/고정/변동) 1카드 + "카테고리 예산 관리 ›" | 카테고리 예산 전체 리스트, 홈 관리 카테고리 피커 |
| 충당금 | 요약 + "관리 ›" | 주머니 편집 (기존 충당금 모달 재사용) |
| 포인트·보상 | "보상 적립" on/off 토글 + "설정 ›" | 보상 적립 대형 폼 전체(기준기간·포인트항목·오늘카드…) |
| 표시 | 테마 세그먼트(간단 → 인라인 유지) | — |
| 연동·데이터 | DayBird / Android 수집 / 정산 규칙 각각 한 줄 행 + 상태 | DayBird 연결 상세, 정산 규칙 폼·리스트 |
| 바로가기 | 검토 대기 · 정산 · 리포트 (§5 Q2) | — |
| 앱 정보 | 버전 · APK 다운로드 (하단 접기) | — |

핵심 이동: **인라인 세부 폼 → drill-in.** 설정 최상위에서는 "무엇이 있고 켜졌는지"만 보이고,
만질 때만 상세로 들어간다.

## 3. 요소 계약 (재구성 후)

| 요소 | 유형 | 동작 | 완료 기준 |
| --- | --- | --- | --- |
| 프로필 행 | 정보 + 액션 | 로그아웃 버튼 | 이메일/이름 표시, 로그아웃 동작 |
| 월 예산 요약 카드 | 정보 | 총/고정/변동 지표 | 이번 달 실제 값 |
| 카테고리 예산 관리 | 내비/모달 열기 | drill-in → 카테고리별 예산 편집 | 편집·저장·취소 동작(현 기능 보존) |
| 홈 관리 카테고리 | 내비/모달 | drill-in → 피커 | 선택 저장이 홈에 반영 |
| 충당금 관리 | 내비/모달 | drill-in | 주머니 CRUD(현 기능 보존) |
| 보상 적립 | 토글 + 설정 | on/off + drill-in 상세 | 토글 즉시 반영, 상세 저장/초기화 |
| 테마 | 세그먼트 | 라이트/다크/시스템 | 즉시 적용 |
| 연동 행(DayBird/Android/정산규칙) | 내비 + 상태 | drill-in | 연결 상태 표시, 상세 진입 |
| 바로가기(검토/정산/리포트) | 내비 | 탭 이동 | §5 Q2 결정에 따름 |
| 앱 정보 | 정보 + 다운로드 | APK 링크 | 버전 표시, 다운로드 동작 |

## 4. 상태 행렬

| 상태 | 설정 목록 | 요약 카드(예산/충당금) | 연동 행 |
| --- | --- | --- | --- |
| 로딩 | 행 스켈레톤 | 값 스켈레톤 | 상태 "확인 중" |
| 데이터 있음 | 전체 행 | 실제 지표 | 연결됨/안됨 배지 |
| 데이터 없음 | 행은 유지 | 0원/미설정 안내 | "연결 안 됨" |
| 오류 | 행 유지 + 재시도 | 이전 값 또는 안내 | 상태 불명 표기 |

## 5. 결정 완료 (§0 참조)

- Q1(drill-in) = **모달**. Q2(바로가기) = **유지**.
- Q3(기능 정리) = DayBird·Android 수집·보상 적립 폼을 **설정 화면에서 제거**(기능/데이터 보존).
  오늘 카드는 별도 '홈 오늘 카드' 구상 세션으로 분리.

## 6. 허용되지 않는 동작

- 세부 튜닝 폼(보상 적립 등)을 설정 최상위에 인라인으로 펼치기.
- 요약 카드·조밀 폼·네비 행을 위계 없이 섞기.
- drill-in 상세에서 기존 기능(예산 편집·충당금 CRUD·정산 규칙)을 누락/파손하기.
