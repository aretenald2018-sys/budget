# 거래 탭 검토 배지 안내 모달 계획

## 요청

거래 탭 월 요약 카드의 `검토 N건 필요` 배지를 누르면 모달을 띄워, 어떤 거래를 어떻게 분류해야 하는지 알려준다.

## 그릴 결과

- 핵심 질문: `검토 N건 필요`은 어떤 화면의 어떤 배지인가?
- 확인: 첨부 스크린샷과 코드상 `render-tx.js`의 `#tx-hero-summary` 안 `pace` 영역에서 `검토 N건 필요` 문구가 렌더링된다.
- 결정: 해당 배지를 실제 `button`으로 렌더링하고, 현재 월의 검토 대상 거래를 요약하는 안내 모달을 연다.
- 남은 가정: 분류 완료는 기존 `검토` 탭 또는 거래 상세 모달에서 처리하고, 이번 작업은 안내와 이동/상세 진입까지만 제공한다.

## 실행 슬라이스

### 슬라이스 1: 거래 탭 검토 배지 안내 모달

- 상태: 실행 완료
- 범위:
  - `render-tx.js`에서 월별 `needsReview` 또는 `needsPaymentRailReview()` 거래 목록을 보관한다.
  - `검토 N건 필요` 배지를 클릭 가능한 버튼으로 바꾸고 안내 모달을 연다.
  - 모달에는 검토 대상 거래 목록과 각 거래별 권장 조치(카테고리 선택, 네이버페이 실제 사용처 보완)를 표시한다.
  - 모달에서 `검토 탭으로 이동`과 개별 거래 `상세` 진입을 제공한다.
  - `styles/20-records.css`에 모달 및 배지 스타일을 추가한다.
  - 프론트엔드 캐시 갱신을 위해 `index.html`, `app.js`, `style.css`의 관련 query string을 갱신한다.
- 수정하지 말 것:
  - 거래 분류 저장 로직 변경
  - Firestore 스키마 변경
  - 기존 검토 탭의 일괄 처리 UX 변경
  - 월 캘린더 집계 방식 변경

## 검증

- `node --check render-tx.js`
- `npm.cmd run verify`
- `git diff --check`
- 실제 확인: 거래 탭에서 `검토 N건 필요` 배지를 눌렀을 때 안내 모달이 열리고, 검토 대상 거래의 조치 문구와 상세 버튼이 보이는지 확인한다.
- 실제 확인: `검토 탭으로 이동` 버튼이 검토 탭으로 이동하는지 확인한다.

## 완료 기준

- `검토 N건 필요` 배지가 버튼처럼 탭 가능하다.
- 모달이 현재 월의 검토 대상 거래 수와 거래별 조치 기준을 보여준다.
- 모달에서 기존 검토 처리 흐름으로 이동할 수 있다.
