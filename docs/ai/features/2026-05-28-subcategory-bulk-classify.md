# 상세분류 미지정 일괄 분류 계획

## 요청

버짓 앱의 카테고리 상세 모달에서 `상세분류 미지정`을 누르면 모달이 뜨고, 리스트에서 선택한 거래들을 선택한 상세분류로 일괄 분류할 수 있게 한다.

## 그릴 결과

- 핵심 질문: `상세분류 미지정` 행은 어디서 동작해야 하는가?
- 확인: 첨부 화면과 코드상 `render-report.js`의 카테고리 거래 모달 안 `subcategorySummaryHtml()`에서 `상세분류 미지정` 요약 행이 렌더링된다.
- 결정: 기존 카테고리 상세 모달 안의 요약 행을 버튼화하고, 같은 `#modals-container`에 중첩 모달을 추가해 미지정 거래만 선택/저장한다.
- 남은 가정: 상세분류 후보는 현재 카테고리의 `subcategories` 설정값을 사용한다. 후보가 없으면 거래 상세 편집에서 먼저 상세분류를 추가해야 한다.

## 실행 슬라이스

### 슬라이스 1: 미지정 거래 일괄 상세분류 지정

- 상태: 실행 완료
- 범위:
  - `render-report.js`에서 `상세분류 미지정` 요약 행 클릭 시 중첩 모달을 연다.
  - 모달에는 현재 카테고리의 미지정 거래 목록, 전체 선택, 상세분류 선택, 저장 버튼을 제공한다.
  - 저장 시 선택된 거래들의 `subcategory`를 `updateTransaction()`으로 갱신하고, 로컬 상태와 기존 카테고리 상세 모달을 새로고침한다.
  - `styles/20-records.css`에 모달/행/액션 영역 스타일을 추가한다.
  - 프론트엔드 캐시 갱신을 위해 `index.html`의 `render-report.js` 버전을 올린다.
- 수정하지 말 것:
  - 카테고리/상세분류 관리 구조 변경
  - 새 Firestore 컬렉션 추가
  - 환급처리 로직 변경
  - 리포트 화면 전체 재설계

## 검증

- `node --check render-report.js`
- `npm.cmd run verify`
- 실제 확인: 로컬에서 앱을 열고 리포트/홈의 카테고리 상세 모달에서 `상세분류 미지정`을 눌러 중첩 모달이 뜨는지 확인한다.
- 실제 확인: 미지정 거래를 1개 이상 선택하고 상세분류 저장 후 요약 행/거래 메타가 선택한 상세분류로 갱신되는지 확인한다.

## 완료 기준

- `상세분류 미지정` 요약 행이 클릭 가능한 컨트롤로 보인다.
- 미지정 거래만 일괄 분류 대상에 포함된다.
- 상세분류를 선택하지 않거나 거래를 선택하지 않으면 저장되지 않고 안내가 뜬다.
- 저장 후 카테고리 상세 모달의 요약과 거래 목록이 갱신된다.
