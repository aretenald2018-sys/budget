# 일일 포인트 룰 다양화와 위젯 루프 기획

## 요청 원문

카드 소비액이 연동된 자동 가계부 기반 앱이다. 과소비 억제를 위해 직전 월의 동일 날짜 소비액보다 당일 소비액이 적으면 그만큼 절약한 금액의 일정 %를 포인트로 적립하고, 특정 금액 달성 시 특정 물품을 살 수 있게 하는 의도로 만들었다. 포인트마다 적립률은 다르다.

단순 억제는 심리학적 반동이 있을 수 있어 목표를 절약이 아니라 적립을 통한 소비로 틀었다. 곧 이 앱을 휴대폰 바탕화면 위젯으로도 만들 예정인데, 단순히 일정 비율을 적립하는 방식은 자율성과 주체성 관점에서 재미가 반감될 수 있고 익숙해지면 억제로 느껴질 여지가 있다. 매일의 관점에서 어떻게 더 즐겁게 목표를 달성하게 만들지, 단순 % 적립을 어떻게 다변화하면 좋을지 기획안을 원한다.

## 이해한 내용

- 목표:
  - 기존 `절약액 * 포인트별 적립률` 구조를 버리지 않고, 매일 사용자가 고르는 작은 놀이 규칙을 얹는다.
  - 앱과 Android 위젯이 "오늘 덜 써야 함"이 아니라 "오늘 어떤 방식으로 포인트를 만들까"를 보여주게 한다.
  - 와인구매, 고급재료, 여행충당 등 포인트별 적립률 차이는 유지한다.
  - 억제, 실패, 벌점, 음수 포인트보다 선택, 회복, 작은 달성, 구체적 소비 보상을 강조한다.
- 비목표:
  - 실제 현금, 계좌 이체, 결제 차감, 포인트 사용 장부까지 이번 기획에 포함하지 않는다.
  - 사행성 룰렛, 실제 결제/현금성 경품, 불투명 랜덤 배율은 넣지 않는다.
  - 다만 사용자가 요청한 가변보상은 확률표, 상한, 하루 1회 공개 조건이 명확한 `오늘의 운 배수`와 시간대 이벤트로 제한해 목업에 반영한다.
  - 의료비, 고정비, 환급 예정액처럼 절약 대상으로 삼으면 안 되는 지출을 게임화하지 않는다.
- 사용자 흐름:
  - 아침 또는 첫 앱/위젯 확인 시 `오늘의 룰 카드` 3장 중 하나를 고른다.
  - 낮에는 위젯에서 오늘 고른 룰, 현재 적립액, 다음 작은 달성까지 남은 금액을 본다.
  - 저녁에는 오늘 만든 포인트와 목표 물품 진행률을 확인하고, 초과 소비일도 `내일 회복 카드`로 이어진다.
- 데이터 가정:
  - 현재 `rewardSavings.pointItems`의 포인트 항목, 적립률, 목표액은 유지한다.
  - 새 일일 룰은 기존 포인트 적립률을 대체하지 않고 `dailyRuleBonus` 또는 `ruleMultiplier`로만 더한다.
  - 일일 선택 기록은 `data.js`를 통해 Firestore에 저장하거나, 1차 MVP에서는 settings 문서의 최근 일자 map으로 제한 저장한다.
- 열려 있는 질문:
  - 이번 HTML 목업에 대한 사용자 피드백을 받은 뒤 실행 슬라이스를 승인한다.
  - 특히 `오늘의 운 배수`가 재미로 느껴지는지, 사행성/피로감으로 느껴지는지 피드백이 필요하다.

## 그릴 결과

- 적용 트리거: `/grill-me`
- 핵심 질문: 매일의 변화를 자동 랜덤 보너스로 줄 것인가, 사용자가 고르는 카드형 룰로 줄 것인가?
- 추천 답변: 사용자가 고르는 카드형 룰. 자율성과 주체성을 살리려면 "오늘 앱이 정해준 배율"보다 "오늘 내가 선택한 공략"이 더 강하다.
- 사용자 답변: 요청 원문에서 자율성, 주체성, 매일의 즐거움, 억제로 느껴질 위험을 명시했다.
- 확정된 결정:
  - 기존 기본 적립률은 신뢰 가능한 기준선으로 유지한다.
  - 매일 3장의 `오늘의 룰 카드`를 제안하고, 사용자는 하나를 선택하거나 기본 적립으로 남을 수 있다.
  - 룰 보너스는 투명한 산식, 낮은 상한, 명확한 대상 포인트를 가진다.
  - 실패 상태는 만들지 않는다. 조건을 못 채운 날은 `기본 적립만 적용` 또는 `오늘은 적립 없음`으로 끝내고, 다음 날 회복 선택지를 보여준다.
- 남은 가정:
  - 첫 버전은 4개 룰 카드만 제공한다. 카드가 많아지면 재미보다 피로가 커질 수 있다.
  - Android 위젯에서는 직접 복잡한 선택 UI를 만들기보다, 선택 전에는 앱 진입 CTA를 보여주고 선택 후에는 상태를 표시한다.

## 목업 피드백 포인트

1. `오늘의 적립` 카드 안에 많은 레이어를 접어 넣는 방식이 괜찮은가, 아니면 `오늘의 룰`을 별도 카드로 분리할 것인가.
2. `오늘의 운 배수`를 매일 공개하는 연출이 즐거운가, 아니면 카드 선택형 보너스만 남기는 편이 더 차분한가.
3. 목표템 갤러리를 홈 첫 화면에 항상 노출할 것인가, 달성 임박 목표만 1개 compact로 노출할 것인가.
4. 위젯은 아침/낮/저녁 자동 상태 전환이 적절한가, 아니면 선택 후 한 가지 compact 상태만 유지할 것인가.

## 서비스 기획안

### 핵심 컨셉

`절약하면 포인트가 쌓인다`에서 `오늘의 공략을 고르면 내 취향의 소비 목표가 가까워진다`로 바꾼다.

기본 산식은 단순하게 유지한다.

```text
기본 절약액 = max(0, 기준 소비액 - 오늘 소비액)
기본 포인트 = 기본 절약액 * 포인트별 적립률
오늘의 보너스 = 룰 카드 조건을 충족한 범위 * 포인트별 보너스율 또는 배율
오늘 적립 = 기본 포인트 + 오늘의 보너스
```

사용자에게는 매일 세 가지 감각을 준다.

- 선택감: 오늘은 와인, 재료, 여행 중 어디에 힘을 줄지 고른다.
- 유능감: `다음 조각까지 3,800원`처럼 오늘 안에 닿을 수 있는 목표를 보여준다.
- 회복감: 초과 소비일 다음 날에도 벌점이 아니라 회복 카드가 열린다.

### 일일 룰 카드

1. `평온 적립`
   - 현재 방식과 동일한 기본 적립.
   - 앱이 피곤한 날에도 사용자가 아무 결정을 하지 않아도 된다.
   - 위젯 문구: `기본 적립 중 · 오늘 +4,200`

2. `포커스 부스트`
   - 사용자가 오늘 집중할 포인트 하나를 고른다.
   - 예: `고급재료 포인트` 선택 시 오늘 절약액 중 첫 20,000원까지 고급재료 보너스 +10%p.
   - 포인트별 기존 적립률 차이를 유지하면서도 "오늘은 재료 쪽으로 몰아주기" 같은 주체성이 생긴다.
   - 위젯 문구: `재료 부스트 ON · 다음 조각 3,800`

3. `카테고리 퀘스트`
   - 앱이 최근 반복 소비 카테고리 중 2-3개를 추천하고 사용자가 하나를 고른다.
   - 예: `카페`, `배달`, `편의점` 중 하나를 오늘 기준선보다 낮게 유지하면 선택 포인트에 소형 보너스.
   - 절대 "금지"가 아니라 "오늘은 이 카테고리에서 포인트 만들기"로 표현한다.
   - 고정비, 병원비, 환급 예정, 필수 교통비는 추천 후보에서 제외한다.

4. `무지출 조각`
   - 하루 전체 무지출이 아니라 사용자가 고른 작은 구간 또는 카테고리를 대상으로 한다.
   - 예: `퇴근 후 간식 0원`, `오후 카페 쉬기`, `배달 쉬기`.
   - 성공하면 작은 고정 보너스 또는 첫 절약액 배율을 준다.
   - 실패해도 패널티는 없고 기본 적립만 남는다.

5. `회복 카드`
   - 전날 또는 이번 달 페이스가 기준선을 넘은 경우에만 등장한다.
   - 오늘 절약액의 첫 구간에 소형 보너스를 붙여 "망했다" 대신 "페이스 복구"를 만든다.
   - 문구는 `어제와 별개로 오늘 6,000원 회복 가능`처럼 날짜를 분리한다.

6. 후순위 `깜짝 제안`
   - 완전 랜덤 배율이 아니라 앱이 세 장의 카드를 섞어 제안하고 사용자가 고르는 방식이다.
   - 같은 카드가 너무 자주 나오지 않게 최근 7일 사용 이력을 반영한다.
   - 투명성을 위해 선택 전 예상 보너스 범위와 상한을 보여준다.

### 위젯 일일 루프

- 선택 전:
  - `오늘의 적립 카드 고르기`
  - `와인 · 재료 · 여행 중 오늘 공략 선택`
  - 탭하면 앱의 홈 카드 선택 영역으로 이동한다.
- 선택 후:
  - `재료 부스트 ON`
  - `오늘 +6,400 · 다음 조각 3,600`
  - 하단에는 `와인 54% · 재료 18% · 여행 9%`처럼 짧게 표시한다.
- 초과 소비 중:
  - `오늘은 기본 적립 없음`
  - `내일 회복 카드 열림`
  - 빨간 경고, 실패, 벌점 표현은 쓰지 않는다.
- 달성 근처:
  - `와인까지 12,800`
  - `오늘 4,000 더 만들면 70%`
  - 목표 물품이 추상 포인트보다 먼저 보이게 한다.

### 화면 구조

- 홈 상단 `오늘의 적립` 카드:
  - 메인 숫자: 오늘 적립 총액.
  - 보조 문구: 선택한 룰 카드와 다음 작은 달성.
  - 포인트별 행: 각 항목의 월 누적, 목표액 진행선, 오늘 보너스 여부.
  - 선택 전에는 카드 3장을 가로 스크롤 또는 compact segmented cards로 보여준다.
- 상세 시트:
  - 오늘의 기준 소비액, 현재 소비액, 기본 절약액, 룰 보너스 산식을 짧게 표시한다.
  - `왜 이 카드가 추천됐나`는 한 줄만 제공한다.
- 설정:
  - 일일 룰 사용 여부.
  - 보너스 상한.
  - 추천 제외 카테고리.
  - 회복 카드 사용 여부.

### 보상 밸런스

- 기본 적립이 전체 보상의 70-90%가 되게 둔다.
- 일일 룰 보너스는 재미와 주체성을 위한 10-30% 보조 장치로 둔다.
- 보너스는 상한을 둔다. 예: 하루 룰 보너스 최대 5,000원 또는 기본 적립의 30%.
- 목표 달성률이 낮은 포인트에 자동으로 보너스를 몰아주는 대신, 사용자에게 `오늘 이쪽에 몰아줄까요?`라고 묻는다.
- 연속 성공 보상은 약하게 둔다. streak가 강하면 다시 억제로 변하므로 `월 3회 쉬어가기` 같은 grace를 제공한다.

### UX 문구 원칙

- 쓰지 않을 표현:
  - `실패`, `과소비`, `초과`, `벌점`, `목표 미달`, `금지`
- 사용할 표현:
  - `오늘은 적립 쉬는 날`
  - `내일 회복 가능`
  - `다음 조각까지`
  - `오늘의 공략`
  - `이 포인트에 몰아주기`
  - `기본 적립 유지`

### 성공 지표

- 위젯 일 1회 이상 확인율.
- `오늘의 룰 카드` 선택률.
- 선택 후 앱 재방문율.
- 목표 포인트 달성까지 남은 일수 감소.
- 초과 소비 다음 날 복귀율.
- 룰 미선택 사용자의 이탈 여부.

## 결정 기록

- 결정: 기본 포인트 적립률은 유지하고, 일일 룰은 보너스 계층으로만 추가한다.
- 이유: 기본 산식이 자주 바뀌면 사용자가 포인트를 신뢰하기 어렵다. 변화는 선택 가능한 보너스에 둔다.
- 되돌릴 수 있는가: 가능. `dailyRewardRules.enabled`를 끄면 기존 포인트 계산으로 돌아간다.

- 결정: 첫 MVP 룰은 `평온 적립`, `포커스 부스트`, `카테고리 퀘스트`, `회복 카드`를 기본으로 하되, 목업 피드백에 따라 `오늘의 운 배수`, `골든아워`, `깜짝 미션`을 같은 카드의 보조 이벤트로 포함할 수 있게 한다.
- 이유: 매일 선택은 재미이지만 과하면 피로가 된다. 사용자 요청의 가변보상은 핵심 방향이므로 독립 규칙이 아니라 보조 레이어로 낮은 강도로 검증한다.
- 되돌릴 수 있는가: 가능. 룰 registry에서 노출 여부만 바꾸면 된다.

- 결정: 위젯은 복잡한 설정을 직접 처리하지 않고, 선택 전 CTA와 선택 후 상태 표시를 담당한다.
- 이유: Android 위젯은 입력 UI 제약이 크고, 복잡한 선택은 앱 안에서 하는 편이 안전하다.
- 되돌릴 수 있는가: 가능. 추후 RemoteViews 버튼으로 빠른 선택을 추가할 수 있다.
