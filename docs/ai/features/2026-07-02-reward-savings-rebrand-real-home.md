### 실제 UI 제약

- 홈의 첫 인상은 이미 `report-hero-card.home-hero-card`가 담당한다. 새 랜딩형 화면을 만들면 안 된다.
- 히어로 카드는 어두운 보라 그라데이션, pill 토글, 흰 글자, 얇은 progress bar를 사용한다.
- 카테고리 분석은 흰색 `budget-gauge-panel` 안에 부모 카테고리와 하위 카테고리 진행바가 반복되는 구조다.
- 하단 nav는 4개 탭만 노출된다. 새 탭을 만들지 말고 기존 홈/목표/거래/검토 흐름 안에 넣는다.
- 현재 검토 카드가 없는 상태도 있다. 검토 대기가 있을 때만 `reviewNudgeCard()`가 끼어드는 기존 조건부 구조를 유지한다.

## 제품 원칙

1. 소진형 언어를 적립형 언어로 바꾼다.
   - 현재: `750,000원 기준 36% 사용`
   - 변경: `이번 2주 47,300원 적립 중`

2. 기존 컴포넌트 패턴을 재사용한다.
   - 새 IA를 만들지 않는다.
   - 실제 홈의 `report-hero-card`, `tds-progress`, 홈 섹션 순서를 유지하되, 사용자가 승인한 이미지 목업의 어두운 카드 톤을 보상 카드에 확대 적용한다.

3. 포인트는 와인/야식 카테고리와 분리한다.
   - `와인/야식`은 실제 결제 기록과 한도 추적이다.
   - `포인트`는 다른 변동비에서 아낀 금액으로 만든 심리적 승인 신호다.
   - 와인 구매 시 실제 거래는 계속 `와인/야식`에 기록하고, 별도 `point_wallet`에서 차감 로그를 남긴다.

4. 실패를 처벌하지 않는다.
   - 기준선보다 더 쓴 날은 `마이너스`가 아니라 `오늘은 적립 없음`으로 표시한다.
   - 빨간 경고, 초과 카운트, streak 리셋을 핵심 화면에 넣지 않는다.

## 핵심 산식

### 기준선

- 대상: 최근 6개월의 변동비/조절비 거래.
- 제외: 고정비, 내 계좌 이동, 투자/저축 이동, 환급 예정, 예산 제외 거래.
- 추천 산출: 주 단위 합계 12~13개를 만든 뒤 중앙값 또는 상하위 10% 트림 평균.
- 파생:
  - 일 기준선 = 주 기준선 / 7
  - 2주 기준선 = 일 기준선 * 14
  - 월 기준선 = 일 기준선 * 해당 월 일수 또는 완료 월 트림 평균

### 세이브포인트

```text
todaySaved = max(0, todayBaseline - todayVariableSpend)
savePoints = todaySaved
```

- 1포인트 = 1원으로 둔다.
- 기준선보다 많이 쓴 날은 `savePoints = 0`.

### 포인트

```text
todayPoints = min(dailyPointCap, todaySaved * pointAllocationRate)
monthPoints = min(120000, sum(todayPoints))
```

- 기본 `pointAllocationRate`: 30%.
- 기본 `dailyPointCap`: 10,000원.
- 월 120,000원은 보장액이 아니라 상한이다.
- 120,000원을 넘는 초과분은 일반 세이브포인트로 남긴다.
- 후속 옵션: 남은 일수와 목표까지 필요한 금액을 매일 재계산하는 적응형 페이싱.

## 기존 홈 통합 IA

### 1단계: 최소 침습

현재 히어로는 그대로 두고, 히어로 바로 아래에 이미지 목업의 어두운 보상 카드 톤으로 `오늘의 적립`을 추가한다.

```text
.report-hero-card.home-hero-card
└─ 기존 "이번 2주 조절비 / 266,890원 / 36% 사용" 유지

오늘의 적립
└─ .budget-gauge-panel
   ├─ 오늘 기준선
   ├─ 오늘 실제 조절비
   ├─ 오늘 적립액 +13,300
   └─ 포인트 +3,990

.home-variable-section
└─ 기존 카테고리 게이지 유지
```

### 2단계: 히어로 프레이밍 반전

히어로의 구조와 스타일은 유지하고 의미만 반전한다.

```text
.report-hero-card.home-hero-card
├─ [이번 2주] [이번 달] [설정] 유지
├─ label: 이번 2주 적립
├─ period: 7/1–7/14 · 2일째 · 조절비 기준
├─ amount: +47,300원
└─ progress:
   ├─ 포인트 14,190 / 120,000
   └─ 절약액 30% 배분 · 월 상한 120,000
```

### 3단계: 카테고리 패널 보강

`home-variable-panel`을 지우지 않는다. 사용자가 적립이 어디서 생겼는지 확인하는 원인 분석 영역으로 재해석한다.

```text
이번 2주 변동비
└─ .budget-gauge-panel.home-variable-panel
   ├─ 포인트 14,190 / 120,000  (새 row, 보상 강조)
   ├─ 생활유지비 13만 / 20만
   ├─ 생활비용 132,050 / 200,000
   └─ 와인/야식 0 / 75,000        (기존 실제 지출 추적 유지)
```

## 데이터 모델

- `daily_baseline`
  - 날짜별 기준 소비액. 첫 구현에서는 실시간 계산으로 시작해도 된다.
- `savings_ledger`
  - `date`, `baseline`, `actualVariableSpend`, `savedAmount`, `savePoints`, `points`, `sourceTxIds`
- `point_wallet`
  - `monthKey`, `balance`, `monthlyCap`, `allocationRate`, `dailyCap`, `debits`
- `streak_log`
  - 선택 기능. 쓰더라도 끊김 페널티는 만들지 않는다.

## 실행 슬라이스

### 슬라이스 1: 계산 뷰모델과 오늘의 적립 카드

- 목표: 기준선/적립액/포인트 계산을 만들고, 홈 히어로 아래에 `오늘의 적립` 카드만 추가한다.
- 예상 수정 파일: `utils/reward-savings.js`, `render-report.js`, `styles/20-records.css` 또는 `styles/60-urge.css`, `app.js`, `index.html`.
- 수정하지 말 것: 히어로 문구 전면 교체, 와인 구매 차감, OS 위젯.
- 검증: 운영 홈에서 기존 히어로가 유지되고 그 아래에 어두운 `오늘의 적립` 카드와 `포인트` 진행바가 보인다.

### 슬라이스 2: 히어로 적립형 전환

- 목표: `report-hero-card.home-hero-card`의 의미를 소진형에서 적립형으로 바꾼다.
- 예상 수정 파일: `render-report.js`, `styles/60-urge.css`, `index.html`, `app.js`.
- 수정하지 말 것: 카테고리 게이지 삭제, 고정비 패널 삭제, 신규 탭 추가.
- 검증: 홈 첫 화면에서 `이번 2주 적립`, `+N원`, `포인트 N / 120,000`이 실제로 보이고 기존 `[이번 2주] [이번 달]` 토글이 유지된다.

### 슬라이스 3: 포인트 폐루프

- 목표: 와인 구매 거래는 `와인/야식`에 남기고, 동시에 `point_wallet` 차감 이력을 남긴다.
- 예상 수정 파일: `data.js`, `render-report.js`, 거래 상세/수정 모달 관련 파일.
- 수정하지 말 것: 결제 자동화, 실제 계좌 이체.
- 검증: 와인 거래를 기록하면 카테고리 지출과 포인트 차감이 서로 맞는다.

### 슬라이스 4: 위젯 요약 API

- 목표: 홈스크린 위젯이 읽을 수 있는 요약 JSON을 만든다.
- 예상 수정 파일: `api/widget-summary.js` 또는 GitHub Pages 제약을 고려한 static/server 경로, 관련 문서.
- 수정하지 말 것: OS별 위젯 구현 전체.
- 검증: 인증된 요약 응답에 `points`, `pointCap`, `todaySavePoints`, `monthSavePoints`가 포함된다.
