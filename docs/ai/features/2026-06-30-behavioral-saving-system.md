# 내 계좌 송금 제외 및 행동 기반 절약 코칭 시스템 계획

## 요청

- 투자/저축을 위해 나의 다른 계좌로 송금한 내역은 지출에서 제외한다.
- 소비내역 분석 결과를 바탕으로 절약 제안/권고를 제공한다.
- 제안은 "와인을 마시지 말라", "식재료를 아껴라" 같은 표면적 문구가 아니라, 임상심리 원리를 참고해 실제 행동이 달라지도록 앱을 시스템화하는 방향이어야 한다.

## 코드 확인 결과

- `data.js`의 `aggregateByCategory()`와 `aggregateMonthlyTotals()`는 `internal_transfer`와 `isBudgetExcluded(tx)`를 지출 집계에서 제외한다.
- `render-report.js`의 `expenseTransactions()`는 `card_payment`, `transfer_out`을 소비 거래로 보고, `isBudgetExcluded(tx)`만 제외한다.
- `api/_lib/server-parser.js`와 `client-parse.js`의 파서 규칙은 은행 출금 송금을 기본적으로 `transfer_out`으로 만든다.
- `render-report.js`에는 환급 예정 거래를 `excludedFromBudget: true`, `excludeReason: 'reimbursement_expected'`로 분리하는 선례가 있다.
- 기존 앱에는 `finance` 목표, `homeManagedCategoryIds`, `mindbank`, `urge`, `pacts`, `cart` 기능이 있어 새 코칭 시스템을 완전히 새로 만들기보다 기존 행동 유도 기능 위에 얹을 수 있다.
- 기존 진단 `docs/ai/diagnoses/2026-05-31-calendar-ingest-gap.md`는 `토스증권`, `네이버페이충전`, 본인 계좌 간 이동이 예산 지출을 부풀리는 문제를 이미 확인했다.

## `/grill-me`

### 핵심 질문

내 계좌 송금은 어떻게 판정할 것인가?

### 추천 답변

초기 버전은 "설정에서 등록한 내 계좌/투자처/충전처 키워드 allowlist"를 기준으로 자동 제외하고, 확신이 낮은 송금은 검토 큐에 올린다.

### 결정

- 추천 답변을 기본 설계로 삼는다.
- 자동 제외 대상은 `토스증권`, `네이버페이충전`, 사용자가 등록한 본인 계좌 별칭/은행/마지막 4자리/수취인 키워드로 시작한다.
- 사람 이름 송금은 본인 계좌 키워드와 명확히 매칭되지 않으면 자동 제외하지 않는다.
- 투자/저축 송금은 지출에서 제외하되 숨기지 않는다. 홈과 리포트에서는 `자산 이동`, `저축 실행`, `투자 실행`으로 별도 표시한다.

### 남은 가정

- 사용자는 내 계좌/투자처 키워드 몇 개를 설정에 등록할 수 있다.
- 과거 거래 자동 일괄 수정은 별도 확인 후 실행한다. 첫 구현에서는 새 유입 거래와 수동 재분류 흐름을 우선한다.
- 임상심리 기반 코칭은 진단/치료가 아니라 행동 설계 원리 적용으로 한정한다.

## 제품 원칙

1. 회계 레이어와 행동 레이어를 분리한다.
   - 회계 레이어: 소비, 수입, 환급/정산, 자산 이동을 정확히 나눈다.
   - 행동 레이어: 소비를 줄일 "다음 행동"을 작게 설계한다.

2. 죄책감 대신 마찰을 설계한다.
   - 초과 지출을 비난하지 않고, 다음 선택 전에 20-90초짜리 멈춤 장치를 둔다.
   - "사지 마세요"가 아니라 "같은 만족을 더 싼 방식으로 얻기", "시간을 미루기", "허용 구매로 기록하기"를 함께 제공한다.

3. 추상 권고보다 상황-행동 계획을 만든다.
   - "카페 줄이기" 대신 "평일 14-17시에 카페 결제 욕구가 뜨면 먼저 사무실 커피 또는 3,000원 이하 메뉴를 선택한다"처럼 `if-then` 형태로 저장한다.

4. 자기통제 실패를 복구 루틴으로 다룬다.
   - 초과 소비가 생긴 날에는 앱이 비난 대신 `복구 선택지`를 제안한다.
   - 예: 오늘 추가 절약 하나, 같은 카테고리 다음 구매 대기, 이번 주 목표 재조정.

5. 사용자의 즐거움을 없애지 않는다.
   - 와인/외식/취미 같은 보상성 소비는 금지가 아니라 "계획된 즐거움"과 "충동성 소비"를 분리한다.
   - 앱은 즐거움을 예산 안으로 옮기는 방향으로 개입한다.

## 실행 슬라이스

### 슬라이스 1: 내 계좌/투자/저축 송금 지출 제외

- 데이터 모델
  - 거래에 `excludeReason: 'own_account_transfer' | 'investment_transfer' | 'savings_transfer' | 'wallet_topup'`를 허용한다.
  - 필요 시 `assetMoveKind: 'internal' | 'investment' | 'savings' | 'wallet_topup'`를 추가한다.
  - 자동 판정 거래에는 `excludedFromBudget: true`를 저장하되, 원본 타입은 검토 가능하도록 보존하거나 `internal_transfer`로 표준화한다.
- 설정
  - 내 계좌/투자처 키워드 목록을 설정에 추가한다.
  - 초기 추천 키워드: `토스증권`, `네이버페이충전`.
  - 사용자가 거래 상세에서 "내 계좌/저축/투자 송금"으로 표시하면 해당 수취인 키워드를 학습 후보로 제안한다.
- 파서/저장 경로
  - `api/_lib/server-parser.js`, `client-parse.js`, `api/_lib/auto-ingest.js` 저장 직전에 공통 판정 helper를 적용한다.
  - `transfer_out` 중 수취인/본문이 내 계좌 키워드와 매칭되면 예산 제외 처리한다.
  - 확신이 낮은 송금은 `needsReview: true`로 남긴다.
- UI
  - 거래 상세 또는 리포트 상세에 `지출에서 제외`, `내 계좌 송금`, `투자/저축 송금` 액션을 둔다.
  - 리포트에는 제외 금액을 "소비에서 빠진 자산 이동"으로 별도 표시한다.
- 제외
  - 소비 코칭 UI는 이 슬라이스에서 구현하지 않는다.
  - 과거 전체 거래 자동 마이그레이션은 별도 슬라이스로 둔다.
- 검증
  - 토스증권 송금과 네이버페이충전 샘플이 월 지출 합계에서 빠진다.
  - 실제 타인 송금은 자동 제외되지 않는다.
  - `aggregateMonthlyTotals()`와 `aggregateByCategory()` 결과에서 제외가 반영된다.
  - 거래 상세에서 수동 제외/복귀가 가능하다.

### 슬라이스 2: 행동 패턴 분석 모델

- 소비 거래를 시간, 요일, 카테고리, 금액, 빈도, 반복 가맹점, 예산 대비 진행률로 요약한다.
- `위험 카테고리`는 금액만이 아니라 다음 신호를 함께 본다.
  - 짧은 기간 반복 결제
  - 늦은 시간대 결제
  - 월급일/스트레스 시점 이후 증가
  - 계획된 즐거움이 아닌 즉시성 구매
  - 예산 초과 직전의 작은 반복 소비
- 출력은 진단명이 아니라 `패턴 가설`로 표현한다.
  - 예: "밤 시간 보상성 소비가 늘어나는 패턴", "소액 반복이 예산을 잠식하는 패턴", "저축 직후 보상 구매가 붙는 패턴"
- 제외
  - LLM 기반 심리 진단, 병리화 표현, 성격 평가를 하지 않는다.
- 검증
  - 같은 월 거래에서 자산 이동 제외 전/후 소비 인사이트가 달라진다.
  - 최소 3개 이상의 패턴 후보가 근거 거래와 함께 표시된다.

### 슬라이스 3: 개인화 행동 개입 홈 카드

- 홈 첫 화면에 "이번 주 하나만 바꿀 행동"을 노출한다.
- 카드 구조
  - 관찰: 최근 패턴 1줄
  - 해석: 어떤 욕구/상황에서 반복되는지 1줄
  - 행동: 오늘 할 수 있는 작은 행동 1개
  - 대체: 같은 만족을 주는 낮은 비용 선택 1개
  - 허용: 계획된 구매로 처리하는 선택 1개
- 예시
  - "이번 주에는 금지보다 지연을 목표로 둡니다. 21시 이후 와인/야식 욕구가 생기면 바로 결제하지 않고 `끌림 들여다보기`에 90초 기록한 뒤, 그래도 원하면 절반 용량/낮은 가격 선택으로 허용합니다."
- 기존 `urge`, `mindbank`, `pacts`, `cart`와 연결한다.
- 제외
  - 모든 카테고리별 장문 조언 생성은 하지 않는다. 한 번에 하나의 행동만 제안한다.
- 검증
  - 홈에서 위험 패턴 하나가 선택되고, 행동 버튼이 기존 탭으로 이어진다.
  - 초과 상태에서도 비난형 문구가 나오지 않는다.

### 슬라이스 4: 주간 회고와 실험 루프

- 매주 한 가지 행동 실험만 고르게 한다.
- 앱은 절약액보다 `실행률`, `지연 성공`, `대체 선택`, `계획 구매 전환`을 함께 보여준다.
- 실패한 주에는 목표를 낮추거나 환경을 바꾸는 질문을 제안한다.
  - "의지가 부족했나요?"가 아니라 "마찰을 어디에 둘까요?"라고 묻는다.
- 검증
  - 주간 회고에서 성공/부분 성공/실패를 기록할 수 있다.
  - 다음 주 홈 카드가 회고 결과를 반영한다.

## 사용자에게 적용할 권고

- 먼저 소비 분석에서 투자/저축/충전 이동을 제거해야 한다. 현재 지출 합계가 부풀면 앱은 잘못된 죄책감과 잘못된 권고를 만든다.
- 절약의 1차 목표는 "쾌락 제거"가 아니라 "충동 구매를 계획 구매로 바꾸기"다.
- 앱은 고위험 순간에만 짧게 개입해야 한다. 평상시에는 숫자를 보여주고, 위험 순간에는 선택 구조를 바꾼다.
- 권고 문구는 카테고리 금지가 아니라 다음 네 가지 중 하나로 제한한다.
  - 지연: 지금 말고 정해진 시간 뒤 다시 결정
  - 축소: 같은 카테고리에서 작은 단위/낮은 가격 선택
  - 대체: 같은 심리적 기능을 하는 행동 선택
  - 허용: 계획된 구매로 인정하고 이후 복구 루틴 연결
- 저축/투자 송금은 소비 실패가 아니라 정체성 강화 피드백으로 표시한다.
  - 예: "이번 달 저축 실행 285,900원. 소비가 아니라 미래 계좌로 이동한 금액입니다."

## 완료 기준

- 내 계좌/투자/저축 송금이 지출 합계와 카테고리 소비 분석에서 제외된다.
- 제외된 금액은 숨겨지지 않고 자산 이동으로 따로 확인된다.
- 앱의 절약 제안은 카테고리 금지 문구가 아니라 사용자 상황에 맞춘 `if-then` 행동으로 제시된다.
- 홈/리포트/끌림/선택 탭이 하나의 행동 루프로 연결된다.

## 다음 실행 시작점

사용자가 이 방향을 승인하면 `docs/ai/features/2026-06-30-behavioral-saving-system.md`의 슬라이스 1만 실행한다. 앱 코칭 UI나 주간 회고는 함께 구현하지 말고, 먼저 내 계좌/투자/저축 송금이 지출에서 제외되는 회계 레이어를 검증한다.
