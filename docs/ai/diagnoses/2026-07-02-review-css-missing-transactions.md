### 첨부 사진 2 - 앱 검토 탭 CSS

- `자동 처리 실패` hero 카드는 배경만 적용되고 내부 글자 대비/크기/간격이 깨져 보인다.
- `전체/거래/원문/영수증` 필터 버튼이 기본 브라우저 버튼처럼 보인다.
- `검토 컨텍스트` 영역의 텍스트가 과도하게 크고 카드 스타일 없이 노출된다.
- 거래 카드 자체는 어느 정도 스타일이 적용되어 있어 전체 CSS 미로딩보다는 검토 탭 전용 선택자 누락에 가깝다.

## 재현/확인

1. `render-review.js`는 현재 다음 마크업을 만든다.
   - `<section class="hero review-hero">`
   - `<div class="chips"> ... <button class="chip">`
   - `<div class="insight review">`
2. CSS 검색 결과:
   - `.review-hero`, `.review-card`, `.review-actions`는 `styles/50-cart-detail.css`에 있다.
   - `.chips`, `.chip`, `.insight.review` 전용 스타일은 현재 검색 결과에서 충분히 확인되지 않았다.
3. 운영 Firestore의 2026-07-02 거래 대조를 읽기 전용으로 시도했으나 현재 Codex 터미널에는 `FIREBASE_SERVICE_ACCOUNT`가 없어 실패했다.
   - 실패: `FIREBASE_SERVICE_ACCOUNT env 미설정`

## 가설

1. 선택 탭 제거 또는 CSS 정리 과정에서 검토 탭이 쓰던 공용 `.chips`/`.chip`/`.insight` 스타일이 함께 사라졌다.
   - 가능성 높음. 증상이 기본 버튼/텍스트 스타일 노출과 일치한다.
2. 검토 탭 hero가 `.hero`와 `.review-hero` 조합을 쓰지만 `.hero` 기본 스타일이 현재 검토 탭에 충분히 남아 있지 않다.
   - 가능성 중간. 배경은 적용되지만 typography와 spacing이 부족해 보인다.
3. 캐시 버스트가 갱신되지 않아 배포본에서 오래된 CSS를 보고 있다.
   - 가능성 중간. 코드 수정 시 `index.html`과 `style.css` query 갱신이 필요하다.
4. 누락 거래는 raw ingest 실패 또는 네이버페이/카드 문자 누락 때문에 Firestore에 transaction이 없다.
   - 가능성 높음. 사용자가 토스 화면과 앱 검토 탭을 함께 제시했다.
5. 일부 후보는 누락 거래가 아니라 소비 제외/취소/자기이체 대상이다.
   - 가능성 높음. `토스증권`, `T맵주차 0원`, `티머니 충전`은 일반 카드 소비와 다르게 처리해야 한다.

## 결론

- CSS 수정은 `render-review.js`의 현 마크업에 맞춰 검토 탭 전용 스타일을 복구하는 작업이다.
- 거래 등록은 사진 기반 수기 생성만으로 끝내면 중복 위험이 있으므로, 먼저 같은 일자/금액/가맹점 대조 후 없는 건만 생성해야 한다.
- 현재 Codex 환경에는 운영 Firestore 쓰기/읽기 비밀값이 없어 실제 등록 검증은 일반 터미널에서 `FIREBASE_SERVICE_ACCOUNT`, `USER_UID`를 가진 상태로 수행해야 한다.

## 제외/주의

- raw message는 삭제하지 않는다.
- Gemini API key, `INGEST_TOKEN`, `FIREBASE_SERVICE_ACCOUNT`를 브라우저 코드나 localStorage에 넣지 않는다.
- `T맵주차` 0원/취소선 거래는 raw 또는 사용자의 명시 확인 없이 소비 거래로 만들지 않는다.
- `토스증권` 이체는 소비 합계에 넣지 않는다.
