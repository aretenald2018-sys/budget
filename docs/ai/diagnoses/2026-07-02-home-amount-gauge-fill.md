# 홈 금액 기준 및 변동비 게이지 색칠 회귀 진단

## 증상

- 홈 hero에서 `이번 달`을 선택했을 때 큰 금액은 `266,890원`인데, 보조 지표의 `이번 달 전체 지출`은 `376,890원`으로 보여 금액 기준이 달라 보인다.
- `이번 달 변동비` 목록에서 사용 금액이 있는 항목도 게이지 트랙이 회색으로만 보이고 사용 비율만큼 색칠되지 않는다.

## 확인한 원인

- `render-report.js`는 홈 hero의 큰 금액을 `controlCategories` 기준으로 계산한다. 이 기준은 `budgetRhythm === fixed`인 고정비를 제외한다.
- 같은 hero의 보조 지표는 `budgetCategories` 전체 기준으로 계산한다. 그래서 `376,890원 - 266,890원 = 110,000원`은 고정비 포함분이다.
- 따라서 두 금액은 같은 범위의 숫자가 아니며, 현재 데이터 기준으로 다른 것이 정상이다. 다만 보조 지표 라벨이 `이번 달 전체 지출`만 말해 `조절비`와의 차이를 충분히 설명하지 못한다.
- `render-report.js`의 변동비 행은 `.gauge-fill` span에 `style="width:N%"`를 넣는다.
- `styles/60-urge.css`에는 `.gauge-fill`의 `display:block`, `height:100%`가 없어 span의 width/height가 트랙을 채우지 못한다.

## 수정 방향

- 보조 지표 라벨을 `고정비 포함 전체 지출`로 바꿔 큰 금액과 범위가 다름을 명확히 한다.
- `.gauge-fill`과 `.tds-progress-fill`에 실제 트랙을 채우는 박스 스타일을 추가한다.
- 데이터 집계 정책은 변경하지 않는다.
