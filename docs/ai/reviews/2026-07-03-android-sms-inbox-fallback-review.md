# Android 문자함 fallback 수집 리뷰

## 결론

- 발견된 차단 이슈 없음.
- 기존 알림 리스너 경로와 신규 SMS inbox 스캔 경로가 같은 로컬 큐 계약을 공유한다.
- 실기기 S24 검증은 ADB 연결 해제 때문에 미완료다.

## 확인한 위험

- foreground 메시지 앱은 알림을 만들지 않아 리스너가 볼 수 없다.
  - 대응: `READ_SMS` 권한이 있으면 앱 진입/복귀/권한 승인 직후 inbox를 직접 스캔한다.
- 잔액/누적 금액을 거래 금액으로 오인할 수 있다.
  - 대응: 금액 바로 앞 라벨이 `잔액`, `누적` 등일 때만 제외하고, `출금55,000원 ... 잔액-55,279,562원`은 55,000원을 잡도록 검증했다.
- 괄호가 있는 가맹점명이 잘릴 수 있다.
  - 대응: after-amount merchant 추출에서 괄호를 분리자로 쓰지 않도록 바꿔 `쿠팡(쿠페이)`를 유지했다.
- 네이버페이 금액 앞 공백이 없을 수 있다.
  - 대응: 네이버페이 결제/취소 정규식에서 금액 앞 공백을 선택으로 처리했다.

## 검증 결과

- `npm.cmd run verify:android-notification`: pass
- `npm.cmd run verify`: pass
- `npm.cmd run pages:build`: pass

## 남은 리스크

- S24가 `adb devices`에서 사라져 v2.0.8 설치와 실제 문자함 로그 확인은 아직 완료되지 않았다.
